Day_51 JDBC03

Jdbc连接的六个步骤，O-R mapping(object relationship mapping 对象关系映射)，DAO(data access object数据访问对象)，java日期类型的转换

一、Jdbc六个步骤（复习）

  1. 六个步骤

    • 1、注册Driver
    • 2、获取连接（url username password）
    • 3、创建PreparedStatement传送工具（提供sql命令）
    • 4、执行sql（完整的sql–>pstm执行；半成品sql–>pstm.setXxxx()–>pstm执行 executeUpdate()–>int executeQuery()–>ResultSet）
    • 处理结果集（next() getXxx(字段下标)）
    • 释放资源 rs–>pstm–>conn

  2. JdbcUtil

    • 函数的封装 配置文件（properties xml）

二、==O-R mapping（object relationship mapping对象关系映射）==

  1. 关系图解：

    从数据库中获取的数据 放在ResultSet（结果集中）,为了方便对于结果集的处理 将结果集中的每一条信息都保存在 一个对象中（对象需要实现序列化，为了以后使用对象流对文件进行读写） ==数据的每一个字段 ——>对象的每一种属性==。

三、==DAO（data access object）数据访问对象==

  1. 作用

    完成对数据库一张表的所有增删查改操作

  2. 命名规范

    表名 + DAO

  3. Dao的数量

    一般一张表对应一个Dao程序

  4. 构成

    常规的五个方法

四、java中的日期类型转换

  1. 进行日期转换的原因

    • 在java程序中，所有的日期类都是java.util.Date类型， 在数据库中 存储日期的类型是java.sql.Date 而通常情况下，用户输入日期 都是以字符串的形式输入的

  2. 字符串——>java.util.Date

  3. java.util.Date——>字符串

  4. java.util.Date——>java.sql.Date

Day_51 JDBC02

编写JDBC程序

一、jdbc程序的创建过程

  1. ==注册Driver——加载Driver驱动类==

    • 因为JVM在使用一个类的时候 需要获取 哪个类的“类对象”，所以需要先使用反射的方法，获取Driver类的“类对象”

      Class.forName(“oracle.jdbc.OracleDriver”)

    • 使用是可以同时注册多个Driver

  2. 获取连接Connection

    • conn = DriverManger.getConnection(地址、用户名、密码)

      实际负责建立数据连接的是Driver

      DriverManager类是驱动的管理者，调用他的静态方法getConnection，从而获取一个Connection对象的对象

    • 地址的组成结构（url）

  3. 创建Statement——传送工具

    • jdbc实际是把用户定义的sql语句，传送到数据库中，然后数据库执行对应的sql语句，Statement就相当于传送工具，将sql语句传输给数据库

      Statment stm = conn.createStatement();

  4. 执行sql命令

    • ==executeUpdate(sql)== : 主要用来执行 insert update delete 命令 , 方法有一个整数类型的返回值( 代表命令执行后,影响了表里的几行数据 )
    • ==executeQuery(sql)== : 主要用来执行 select 命令 , 方法返回值为ResultSet(里面封装了查询结果数据 )
    • execute(sql) : 可以执行所有sql ( 特别是DDL命令 ) , 返回一个boolean值( 是否有结果集返回 )

  5. 处理结果集

    • 当sql命令是select时，因为需要返回查询的结果，所以需要一个结果及（ResultSet）用来接收查询到的结果

  6. 释放资源——按照资源创建的顺序反向释放

    ResultSet——>Statement——>Connection

二、带参的sql命令的执行==（PreparedStatement）==

  1. 使用Statement 执行sql命令的不足

    • 进行字符串的拼接较为麻烦

    • sql注入 可能引起数据资源的不正当获取（使数据不安全）

  2. 使用PreparedStatement执行sql命令 — Statement的子接口

    • 创建PreparedStatement对象

      注：pstm允许用户给定的sql是一个“半成品”，使用“？”补全sql语句，后续使用时，需要用户指定“？”指代的值

    • 为“？”设置值 ——pstm提供了一些列的==setXxx(..)==方法

      ​ pstm.setInt( p1,p2) pstm.setString( p1,p2 )

      ​ pstm.setDouble(p1,p2) pstm.setDate(p1,p2)

      ​ P1 — ?的下标(从1开始) p2 — ?的值

      ​ 注意: setDate时值的类型为java.sql.Date

    • 执行sql命令

      executeUpdate() 增、删、改

      executeQuery() 查 结果会返回一个ResultSet

  3. PreparedStatement与Statement的区别

    功能 区别
    构建 stm –> conn.createStatement() 不需要指定sql Pstm –> conn.prepareStatement(sql) 半成品都行
    执行 stm.executeXxxxx( sql ); pstm.executeXxxx( );
    选择 ① 单条命令执行 : 带参sql选择pstm – 安全 ② 多条”同构sql” : 选择pstm效率高( 预编译 )

三、JDBCUtil工具类的封装

  1. 代码封装的基本套路

    • 能够完成一定功能,并且多次出现在程序中的一段代码,需要被封装成函数,提高代码复用性
    • 对于程序中可能会变化的数据, 将其定义在文本文件(配置文件)中 , 程序里通过io获取文件数据

  2. 配置文件（纯文本文件）

    • 常规文件类型 : xxx.properties文件 xxx.xml文件、properties文件格式 — key=value 每一行只存一个数据

    • 集合类Properties — 专业处理properties文件的工具

      ​ ① key和value都是String

      ​ ② 提供了函数 load( InputStream – 读配置文件的流对象 ) 获取配置文件信息

      ​ ③ map.getProperty( key ) — value

  3. 文件流的处理

    • ① Class类提供了getResourceAsStream() , 可以帮助获取一个读文件的字节输入流

      ② 方法需要的路径为”虚拟路径” — 从 ”/” 开始写 , 代表当前环境的Classpath在运行时,

      会自动获取当前环境的classpath,替换掉“/”

Day_50 JDBC

jdbc的概念、导入到myeclipse

一、引言：

  • 概念：java语言连接数据库的桥梁

  • 组成

    • 接口：由sun定义，提供给程序员开发使用，通过java.sql包和javax.sql包 给出

    • 实现类：由数据库厂商提供，需要配置到开发环境里，通常以xxx.jar包的形式提供

      ​ jdk—6或更早 使用ojdbc5.jar jak—7或更新 使用ojdbc6.jar

  • 搭建开发环境

    *<将jar包配置到开发环境>*

    • 使用的是普通的文本编译器 ——将jar包所在路径配置到环境变量CLASSPATH中
    • 使用的是集成开发工具 myeclipse ——以project管理代码
      • 在当前project下面新建一个lib文件夹
      • 将jar拷贝到文件夹下
      • 右键选中 jar->add to build path
    • 在myeclipse里配置执行sql命令的窗口
      • windows ->show view ->other ->搜索 db browse ->选中
      • 窗口里点击右键->new connection
      • 在项目里新建sql包，包里面新建xxx.sql文件，在文件中可以编写执行sql命令，默认事务为一条一提交（自动补充commit）

业务层、视图层、三层结构、连接池

一、Review

  1. 实体类的封装 ——entity

    • 作用：用来封装数据库中表里的数据（通过属性来保存表里相应字段的数据）
    • 要求：属性私有 get 和 set 无参构造 实现Serializable

  2. O-R Mapping ——DAO

    • ==db 与 java 里的对应关系==

      数据库 java
      Table class
      字段 属性
      行 对象

    • mapping在代码上的体现 ——DAO

      1. DAO：数据访问对象——完成对数据库一张表的增删查改

      2. 步骤：①定义Dao接口（给出方法的声明与定义）

        ​ ②选择特定的技术实现接口的定义（JDBC）

      3. ==常规方法：增、删、更新、查一个、查全部==

        方法 实现过程
        insert 参数是实体对象（没有ID），实现时先补齐参数（给ID属性赋值），再插入到数据库
        update 参数是对象（完整，属性可能会被更改），实现是需要对整行数据进行修改（除了id）
        delete 参数只有主属性（不要掺杂其他属性）
        selectOne 参数为id
        selectAll 返回值为List

      4. 结果：屏蔽数据库的差异

二、业务层（Service层，biz层）

  1. 作用

    通过定义函数来完成用户实际的业务需求（最终业务的实现还是会落实到数据库的访问操作Dao）

  2. 业务的其他作用

    除了完成业务需求，还要控制事务

  3. Jdbc里的事务处理

    默认为一条一提交（程序会默认再每一条命令后自动补全commit）

  4. ==Jdbc里手动控制事务的API==

    ①方法

    ​ 设置事务提交策略：conn.setAutoCommit(boolean); true：自动 、false：手动

    ​ 提交：conn.commit();

    ​ 回滚：conn.rollback()；

    ​ 注意：事务提交策略一经设置，永久生效（直到conn消亡）

    ②代码结构

    ​

  5. 如何将事务控制应用到现有的业务层代码中

    ①方式一：参数传递（造成接口污染）

    ​ <接口污染就是：会强制限制Dao层连接数据库的技术要求>

    ②方式二：借助ThreadLocal类完成工作

    • 作用：线程局部变量

      Ⅰ、一个ThreadLocal对象可以为同一个线程绑定一个”独占资源“，使得再这个线程体的任意代码部分都可以获取这个数据

      Ⅱ、==可以为不同的线程绑定不同的数据==

    • 使用步骤

      操作流程
      Ⅰ、创建ThreadLocal对象，并且在创建过程中通过<泛型>指定绑定数据的类型 ThreadLocal<数据类型> tol = new ThreadLocal<数据类型>();
      Ⅱ、为线程绑定数据 tol.set(数据);
      Ⅲ：获取数据 tol.get(数据) 获取当前绑定的线程的数据
      Ⅳ：解除绑定：tol.remove()

    • 在实际中的应用

      业务功能代码分散在多个代码块中，但是属于同一个线程体的内容

      使用时：我们希望同一个线程只能获取一个连接对象

      ​ 不同的线程获取不同的连接对象

      对于util文件的修改

    • ==tol解除绑定==

      理论上连接应该在线程结束时关闭，实际开发中，因为无法确定线程结束的时间，所以在事务结束时对conn进行关闭，一个事务 会由多个Dao组合成，所以conn一定不能再Dao层中关闭，只能在业务层中关闭conn ==关闭后一定要解除绑定 tol.remove()==（防止多线程的 并发问题）

  6. 业务层代码开发步骤

    ①获取连接，设置提交策略（手动）

    ②完成业务需求（数据的判断，调用dao层方法，业务数据处理）

    ③事务进行commit 或 rollback

    ④关闭资源（业务层只关闭conn， rs 和 pstm在Dao中关闭）
